Robert Game

Designing the Robert game, an immitation of the Qbert game offered numerous learning opportunities in various areas of game development and programming. Here's a are the skills and knowledge taken from the process:

Designing the game

Part of the design of the game was building the game engine itself and some of the skills learned can be split as:

C++ Programming Skills:
Object-Oriented Programming: Using classes and objects to structure the game.
Implementing th use of libraries.
Windows API:
Interacting with the Windows API: Learning to work with Windows-specific functions and data types (HINSTANCE, HWND, HDC).
Event Handling: Understanding Windows messages and handling events like painting and activation.
Game Loop and Frame Rate Management:
Structure a game loop to handle updating game state and rendering frames.
Managing the frame rate to ensure smooth gameplay.
Graphics and Rendering:
Loading and drawing bitmaps, managing different orientations for the game characters.
Game Engine Utilization:
Learned how to initialize, configure, and utilize a game engine.
Setting up game resources like bitmaps and initializing game state
Input Handling:
Handling keyboard input for game controls, including directional movement.
Using placeholder functions for handling mouse input, which could be extended for future features or games.
Collision Detection and Response:
Detecting and handling collisions betwenn bitmaps and sprites to achieve the purpose of the game.
User Interface and Interaction.
Using message boxes to interact with the player.
Managing different game states such as starting, running, and ending.
